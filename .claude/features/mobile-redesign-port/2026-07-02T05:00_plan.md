# Mobile Redesign Port — Implementation Plan

> Guiding principles (from CLAUDE.md Development Strategy + user): **least code, maximum
> reuse, surgical changes, consistent look-and-feel, no regressions.** Every pass is
> independently shippable and guarded by the parity register (research §6).

## 1. Architecture overview

### 1.1 Token flow (the spine of the whole port)

```
 src/index.css  :root { --bg-0..--bg-4, --ink-1..--ink-6, --accent(+derived),
                        --info/--success/--warn/--danger(+ -soft), --border-1/2 }
        │                                   │
        ├── tailwind.config.js  colors:{ 'bg-0':'var(--bg-0)', accent:'var(--accent)', ... }
        │        → ergonomic classes: bg-bg-1, text-ink-2, border-accent, text-danger
        │
        └── src/utils/tokens.ts  (TS mirror for JS-computed inline styles)
                 → tuning colors re-exported from tunings.ts (Palette A, canonical)
                 → avatarColor re-exported from songAvatar.ts
                 → instrument spine, badge tone map
```

One source of truth. A re-skin = edit `:root`. JS and Tailwind both read the same tokens.

### 1.2 Navigation shell (5-tab, mobile-first; desktop = sidebar superset)

```
ProtectedLayoutRoute (unchanged auth gating)
  └── ModernLayout
        ├── Sidebar            (desktop md+, reconciled to same IA, superset)
        ├── MobileHeader       (md-, + NotifBell → /notifications)
        ├── <main>             (Outlet; pb for bottom nav on mobile)
        └── BottomNavigation   (mobile only; re-skinned; Home·Songs·Sets·Calendar·More)

Routes added:  /  (Home dashboard, stop redirect→/songs)
               /calendar   (aggregator: shows+practices+events)
               /more        (hub: Jam, Band, Events, Friends, Settings, Profile, Help)
               /events, /events/:id       (new)
               /friends                    (new)
               /notifications              (new)
Routes moved under More (still top-level paths, just not tabs): /jam, /band-members, /settings
Reached through Calendar: /shows, /practices (paths unchanged)
```

### 1.3 Primitive layer (build the small gaps; reuse the rest)

| Need                    | Source                                                                                                                                                                        | Plan                                                               |
| ----------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------ |
| tokens.ts               | new                                                                                                                                                                           | thin re-export module                                              |
| Badge (static semantic) | new `common/Badge.tsx`                                                                                                                                                        | tone map info/success/warn/danger/neutral/accent                   |
| Eyebrow (mono label)    | new `common/Eyebrow.tsx`                                                                                                                                                      | trivial                                                            |
| BottomSheet             | extend `common/SlideOutTray.tsx` w/ `position:'bottom'`                                                                                                                       | no new component if tray generalizes cleanly                       |
| Avatar (generic)        | generalize `common/SongAvatar.tsx`                                                                                                                                            | accept `{label,color,photo,size}`; keep SongAvatar as thin wrapper |
| everything else         | MetaPill, TuningPill, SectionCard, LinkIcons, MarkdownRenderer, DatePicker, TimePickerDropdown, ConfirmDialog, EmptyState, KebabMenu, TabSwitcher, ToastContext, lucide-react | reuse as-is; only re-point colors to tokens                        |

### 1.4 New-feature backend (all Supabase-only + Realtime; NOT band sync engine)

```
events        migration → EventService     → RealtimeManager(event_participants, event_lineup_requests)
friends       migration → FriendService     → RealtimeManager(friend_requests)
                        + edge fns: resolve-friend-code, send-friend-request, accept-friend-request
notifications migration → NotificationService→ RealtimeManager(notifications)
```

Each new table: idempotent DDL, GRANTs to authenticated + service_role, RLS, publication add +
REPLICA IDENTITY FULL, CHECK constraints matching TS unions exactly (enum-drift is fatal).

## 3. Phased passes (each = its own `/implement` cycle)

- **P0 Token foundation** (UI-only, app-wide). Verify: app builds, renders dark, one orange
  everywhere, no visual regression on a sampled screen. `npm run test:quick` + type-check.
- **P1 Primitives**: tokens.ts, Badge, Eyebrow, BottomSheet, generic Avatar. Unit tests for Badge/Avatar.
- **P2 Nav/shell**: re-skin+wire BottomNavigation, ModernLayout branch, Sidebar IA, new empty routes + header bell placeholder. E2E: tabs navigate, shell persists (parity w/ persistent-layout).
- **P3 Songs re-skin** — guard: SyncIcon 5 states, band/personal tabs, Copy-to-Personal + linked chain, filters/sorts, notes modal (band+personal), EditSongModal depth, LinkIcons popover.
- **P4 Setlists re-skin** — guard: section/break items (+dur/notes), drag reorder+autosave, per-song notes, show-link fork, both editors, band/personal tabs.
- **P5 Shows re-skin** — guard: tap-to-call/mailto contacts, load-in/soundcheck/payment, setlist fork.
- **P6 Practices re-skin (incl. Session)** — guard: play-along 4 layouts + persistence, foot-pedal/keyboard scroll, tuning-change alert, session notes.
- **P7 Band / Settings / Profile re-skin** — guard: regenerate invite, transfer ownership, admin toggles, permission gating, edit instruments.
- **P8 Jam re-skin** — guard: QR, common-songs auto-match, end-session flow, anon view poll.
- **P9 Home dashboard + Calendar aggregator + More hub** (net-new UI over existing data).
- **P10 Notifications** (migration → service → bell/center/digest/whats-new).
- **P11 Friends** (migration + edge fns → service → friends home/add/join-by-code/QR).
- **P12 Events** (migration → service → host/guest lineup, requests/approve, RSVP, access, calendar integration).

P0–P9 + Calendar/Home/More need no DB. P10–P12 are the DB features, independent of the re-skins.

## 4. Testing strategy

- Every pass: `npm run type-check` + `npm run test:quick` before/after; `npm run lint`.
- Migrations (P10–P12): `supabase db reset` + `npm run test:db` (pgTAP per new table:
  columns/constraints/indexes/RLS/grants) + `npm run lint:migrations` gate.
- Edge fns: `./scripts/deploy-edge-functions.sh` + `./scripts/smoke-edge-functions.sh`; row in FUNCTIONS.md.
- New interactive elements get `data-testid` + `id` per CLAUDE.md Testability Standards.
- Parity: each re-skin pass carries its research-§6 checklist as explicit acceptance criteria; E2E
  smoke on the reskinned flow before marking done.
- No native dialogs/scrollbars (use ConfirmDialog/toast, custom-scrollbar) per CLAUDE.md.

## 5. Risks & mitigations

- **65-file hex migration** → do it per-screen inside each pass, never big-bang; P0 only adds tokens
  - unifies orange, does not restructure.
- **Enum drift** (TS vs SQL) → define CHECK constraints identical to TS unions from day one.
- **SyncIcon regression on mobile** → explicit acceptance criterion on P3–P6.
- **Two setlist editors** → P4 must reskin both or consciously converge (flag before converging).
- **Desktop IA** → sidebar kept as superset (decision §9.2); don't remove desktop affordances.
- **Casting** → never delete services/tables; only ensure UI stays unmounted.
