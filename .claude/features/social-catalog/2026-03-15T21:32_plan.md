# Implementation Plan: Social Catalog

## Executive Summary

Three phased capabilities delivered as one cohesive feature:

1. **Phase 1 — Personal Song Catalog** (~8h): Unlock the personal song context in the UI. Schema already supports it; needs RLS updates, `normalize_text()` SQL function, and a "Personal" tab on the Songs page.
2. **Phase 2 — Personal Setlists** (~6h): Relax `setlists.band_id` to nullable, add context columns, unlock personal setlist UI.
3. **Phase 3 — Jam Session Core** (~14h): New tables, matching engine, host/join/view UI, QR codes.
4. **Phase 4 — Anon View + Save** (~8h): Supabase Edge Function, public route, save-as-setlist flow.
5. **Phase 5 — Polish** (~4h): Expiry cleanup, tier stub wiring, navigation, fuzzy match confirmation UI.

Each phase must be complete before the next begins. Phases 1 and 2 unlock independent user value even before jam sessions ship.

## Phase 1: Personal Song Catalog

### Database Changes

**In `supabase/migrations/20251106000000_baseline_schema.sql`:**

1. Add `normalize_text()` SQL function (BEFORE songs table creation so generated columns can reference it):

```sql
CREATE OR REPLACE FUNCTION normalize_text(input TEXT)
RETURNS TEXT LANGUAGE sql IMMUTABLE STRICT AS $$
  SELECT trim(regexp_replace(
    regexp_replace(
      regexp_replace(lower(input), '^(the|a|an)\s+', '', 'i'),
      '[^a-z0-9 ]', '', 'g'
    ), '\s+', ' ', 'g'
  ));
$$;
```

2. Add to `songs` table definition (new columns after `song_group_id`):

```sql
normalized_title TEXT GENERATED ALWAYS AS (normalize_text(title)) STORED,
normalized_artist TEXT GENERATED ALWAYS AS (normalize_text(COALESCE(artist, ''))) STORED,
```

3. Add index after the songs indexes section:

```sql
CREATE INDEX idx_songs_normalized ON public.songs(normalized_title, normalized_artist);
```

4. Update RLS policies — replace `songs_select_band_members_only` with two policies:

```sql
-- Personal songs: owner can see their own
CREATE POLICY "songs_select_personal_own"
  ON public.songs FOR SELECT TO authenticated
  USING (context_type = 'personal' AND context_id = (select auth.uid())::text);

-- Band songs: band members can see
CREATE POLICY "songs_select_band_members"
  ON public.songs FOR SELECT TO authenticated
  USING (context_type = 'band' AND is_band_member(context_id::uuid, (select auth.uid())));
```

Similarly update INSERT/UPDATE/DELETE policies to handle personal context.

## Phase 3: Jam Session Core

### Database Changes

**In baseline migration (add after setlists, before or in Section 2):**

```sql
-- jam_sessions
CREATE TABLE public.jam_sessions (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  short_code TEXT UNIQUE NOT NULL,
  name TEXT,
  host_user_id UUID NOT NULL REFERENCES public.users(id) ON DELETE CASCADE,
  status TEXT NOT NULL DEFAULT 'active'
    CHECK (status IN ('active', 'expired', 'saved')),
  created_date TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  expires_at TIMESTAMPTZ NOT NULL,
  saved_setlist_id UUID REFERENCES public.setlists(id) ON DELETE SET NULL,
  view_token TEXT UNIQUE,
  view_token_expires_at TIMESTAMPTZ,
  settings JSONB DEFAULT '{}'::jsonb,
  version INTEGER DEFAULT 1 NOT NULL,
  last_modified_by UUID REFERENCES public.users(id)
);

-- jam_participants
CREATE TABLE public.jam_participants (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  jam_session_id UUID NOT NULL REFERENCES public.jam_sessions(id) ON DELETE CASCADE,
  user_id UUID NOT NULL REFERENCES public.users(id) ON DELETE CASCADE,
  joined_date TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  status TEXT NOT NULL DEFAULT 'active'
    CHECK (status IN ('active', 'left', 'kicked')),
  shared_contexts JSONB NOT NULL DEFAULT '[]'::jsonb,
  UNIQUE(jam_session_id, user_id)
);

-- jam_song_matches
CREATE TABLE public.jam_song_matches (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  jam_session_id UUID NOT NULL REFERENCES public.jam_sessions(id) ON DELETE CASCADE,
  canonical_title TEXT NOT NULL,
  canonical_artist TEXT NOT NULL,
  display_title TEXT NOT NULL,
  display_artist TEXT NOT NULL,
  match_confidence TEXT NOT NULL DEFAULT 'exact'
    CHECK (match_confidence IN ('exact', 'fuzzy', 'manual')),
  is_confirmed BOOLEAN NOT NULL DEFAULT true,
  matched_songs JSONB NOT NULL DEFAULT '[]'::jsonb,
  participant_count INTEGER NOT NULL DEFAULT 0,
  computed_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
```

### New Service

**`src/services/JamSessionService.ts`:**

```
JamSessionService
  ├── createSession(hostUserId, name?, settings?) → { session, rawViewToken }
  │     - generate short_code (6-char alphanumeric, collision-retry)
  │     - generate raw viewToken (crypto.randomUUID())
  │     - hash viewToken with SHA-256 → store in DB
  │     - set expires_at based on account_tier (stub: always 24h free)
  │
  ├── joinSession(shortCode, userId, sharedContexts) → JamSession
  │     - SELECT jam_sessions WHERE short_code = ? AND status = 'active'
  │     - INSERT jam_participants
  │     - trigger recomputeMatches()
  │
  ├── recomputeMatches(sessionId) → JamSongMatch[]
  │     - fetch all active participants + their shared_contexts
  │     - for each context: getSongs({ contextType, contextId })
  │     - run matching algorithm (normalize → exact → fuzzy)
  │     - DELETE old jam_song_matches for session
  │     - INSERT new matches
  │
  ├── confirmMatch(matchId) → void          (fuzzy → confirmed)
  ├── dismissMatch(matchId) → void          (remove fuzzy match)
  ├── addManualMatch(sessionId, songIdA, songIdB) → JamSongMatch
  │
  ├── saveAsSetlist(sessionId, hostUserId) → Setlist
  │     - collect confirmed matches → SetlistItems
  │     - createPersonalSetlist with tags: ['jam'], jamSessionId
  │     - UPDATE jam_sessions SET status='saved', saved_setlist_id=...
  │
  ├── expireSession(sessionId) → void
  ├── canCreateJamSession(userId) → { allowed, reason? }   (stub: always true)
  └── generateShareUrl(session, rawToken) → string
```

### Save as Setlist Flow

On `JamSessionPage`, "Save as Setlist" button:

1. Calls `JamSessionService.saveAsSetlist(sessionId, userId)`
2. Service creates personal setlist: `{ contextType: 'personal', contextId: userId, tags: ['jam'], jamSessionId: sessionId, name: session.name || 'Jam Session' }`
3. Setlist `items` = confirmed matches converted to `SetlistItem[]` (type='song', songId from matched_songs where userId=currentUserId)
4. Updates `jam_sessions.status = 'saved'` + `saved_setlist_id`
5. Shows toast: "Jam session saved as setlist!"
6. Navigates to `/setlists` (personal tab)

## Risk Mitigation

| Risk                                              | Mitigation                                                                               |
| ------------------------------------------------- | ---------------------------------------------------------------------------------------- |
| Matching false positives                          | Multi-tier algorithm; fuzzy matches require host confirmation; 50+ unit test cases       |
| Personal songs data leak                          | RLS split policy; jam_song_matches never stores raw song rows; service-role compute only |
| setlists.band_id nullable breaks existing queries | Audit RemoteRepository + LocalRepository before migration; add context_check constraint  |
| view_token guessable                              | 122-bit entropy UUID; SHA-256 hashed; expiry checked; Edge Function rate-limited         |
| Realtime non-band channel                         | New `jam:session:{id}` channel pattern; extend RealtimeManager event types               |
| qrcode.react bundle size                          | Lazy-load JamInviteQR component; QR only needed on one page                              |
